# 字帖預處理工具集

這個目錄包含處理不同排版字帖的預處理工具。

## 工具列表

### 1. `crop_and_denoise.py`
**用途**: 處理有紅框的字帖
**功能**:
- 自動偵測紅色邊框
- 物理裁切字帖區域
- 消除浮水印
- 高品質黑白化

**適用場景**: 有紅色邊框標記的字帖圖片

**使用方式**:
```python
# 修改檔案中的路徑設定
BASE_PATH = "你的原始圖片目錄"
CROP_OUTPUT = "裁切輸出目錄"
BW_OUTPUT = "黑白圖輸出目錄"

# 執行
python crop_and_denoise.py
```

---

### 2. `grid_splitter.py`
**用途**: 處理網格式排版字帖（如 4x2 格式）
**功能**:
- 去除格線
- 等比例分割單字
- 自動置中縮放到 512x512

**適用場景**: 格線式排版的字帖（如田字格、米字格）

**參數調整**:
```python
# 在檔案最後修改
run_simple_line_erase_split(
    rows=4,              # 橫排數量
    cols=2,              # 直排數量
    line_thickness=4     # 線條粗細（越大抹除越強）
)
```

---

### 3. `watermark_remover.py`
**用途**: 專門消除浮水印
**功能**:
- 使用中值濾波消除白色浮水印
- 形態學閉運算填補筆畫空洞
- 連通域過濾去除碎片

**適用場景**: 已裁切的字帖圖片，但有浮水印干擾

---

### 4. `full_pipeline.py` ⭐ 推薦
**用途**: 完整的一鍵處理流程
**功能**:
- 自動裁切紅框
- 去浮水印
- 黑白化
- 4x2 網格分割
- 單字置中輸出

**適用場景**: 綜合處理，適合大部分字帖

**重要參數**:
```python
PADDING_V = 25    # 上下留白（避免切到字）
PADDING_H = 10    # 左右留白
ROWS = 4          # 字帖行數
COLS = 2          # 字帖列數
CANVAS_SIZE = 512 # 輸出圖片尺寸
```

---

### 5. `manual_patcher.py`
**用途**: 人工檢查後重新排序
**功能**:
- 將手動補充的字插回正確位置
- 自動重新編號

**使用情境**:
1. 自動分割後發現有錯誤/模糊的字
2. 手動刪除錯誤的圖片
3. 手動補充正確的字（檔名：序號.png，如 `11.png`, `17.png`）
4. 執行此工具重新排序

**使用方式**:
```python
# 設定路徑
AUTO_SPLIT_DIR = "自動分割的圖片目錄"
PATCH_DIR = "手動補充的字目錄"
FINAL_DIR = "最終輸出目錄"

# 執行
smart_patch_sequencer(total_chars=897)  # 總字數
```

## 處理流程建議

### 標準流程
```
原始字帖圖片
    ↓
full_pipeline.py (一鍵處理)
    ↓
人工檢查 & 刪除錯誤圖片
    ↓
手動補充缺失的字 → patch 目錄
    ↓
manual_patcher.py (重新排序)
    ↓
最終字庫完成！
```

### 針對特殊情況
- **只有紅框，無格線**: 使用 `crop_and_denoise.py`
- **只有格線，無紅框**: 使用 `grid_splitter.py`
- **已裁切，只需去浮水印**: 使用 `watermark_remover.py`

---

## 注意事項

1. **備份原始圖片**: 處理前務必備份原始檔案
2. **路徑設定**: 每個工具都需要修改檔案中的路徑設定
3. **參數調整**: 根據實際字帖調整參數（如格線粗細、留白大小）
4. **人工檢查**: 自動處理後務必人工檢查品質

## 開發者備註

這些工具是從實際處理 4 本字帖的經驗中累積而來：
- `00_智永`: 紅框 + 浮水印
- `01_沈尹默`: 不同排版
- `02_顏真卿`: 米字格
- `03_歐陽詢`: 紅框 + 米字格 + 浮水印

每本字帖的排版都不同，因此保留了多個版本的工具以應對不同情況。
